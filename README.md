# FastRead Frontend

React-интерфейс образовательной платформы FastRead для развития скорости чтения, внимания, памяти и понимания текста.

Приложение позволяет проходить курсы и отдельные упражнения, подбирать задания по заданным параметрам, контролировать дневную нагрузку и отслеживать изменение результатов обучения.

> Статус: завершенный учебный проект. Репозиторий содержит клиентскую часть и исходную Django-интеграцию, использовавшуюся для запуска приложения.

## О проекте

FastRead — образовательный сервис для систематических тренировок навыков чтения.

Пользователь может зарегистрироваться, выбрать курс или отдельное упражнение, выполнять задания и отслеживать прогресс по нескольким показателям:

- коэффициент эффективности чтения;
- широта поля зрения;
- визуальная память;
- визуальная память с учетом позиции;
- логическая память;
- коэффициент внимания.

Клиентское приложение отвечает за отображение курсов и упражнений, управление состоянием пользовательской сессии, взаимодействие с REST API и визуализацию результатов.

## Основные возможности

### Авторизация

- регистрация пользователя;
- вход по логину и паролю;
- JWT-аутентификация;
- обновление пользовательской сессии;
- выход из аккаунта;
- обработка ошибок авторизации.

### Курсы

- просмотр каталога курсов;
- сортировка курсов;
- просмотр описания курса;
- отображение модулей и заданий;
- запись пользователя на курс;
- отображение прогресса по каждому модулю;
- переход к следующему незавершенному заданию;
- отображение дневной нормы;
- уведомление о завершении курса.

### Упражнения

- просмотр доступных упражнений;
- подбор упражнений по теме и типу;
- расширенная фильтрация;
- фильтрация по целевой аудитории и методике;
- выбор упражнений по развиваемым показателям;
- прохождение интерактивных заданий;
- сохранение ответов;
- фиксация времени выполнения и результата.

### Профиль пользователя

- просмотр и редактирование личных данных;
- изменение аватара;
- изменение пароля;
- настройка дневной нагрузки;
- управление уведомлениями;
- удаление аккаунта;
- просмотр динамики результатов.

### Аналитика

В профиле отображаются:

- сводная диаграмма текущих показателей;
- история изменения выбранного показателя;
- статистика результатов тренировок;
- прогресс прохождения курсов;
- количество выполненных за день заданий.

## Пользовательский сценарий

1. Пользователь создает аккаунт или входит в систему.
2. Выбирает готовый курс либо подбирает отдельное упражнение.
3. Настраивает параметры и дневную нагрузку.
4. Выполняет интерактивные задания.
5. Система сохраняет ответы, время выполнения и результат.
6. Пользователь просматривает прогресс в личном кабинете.
7. На основании текущих показателей выбираются следующие упражнения.

## Управление состоянием

Глобальное состояние приложения хранится в Redux.

Асинхронные операции выполняются через Redux Thunk:

```text
React component
      │
      ▼
Redux action
      │
      ▼
Thunk
      │
      ▼
Axios request
      │
      ▼
FastRead REST API
      │
      ▼
Reducer
      │
      ▼
Updated UI
```

В Redux хранятся:

- состояние авторизации;
- данные пользователя;
- список курсов;
- текущий курс;
- выбранное упражнение;
- прогресс пользователя;
- результаты запросов;
- сообщения об ошибках и успешных операциях.

## Локальная сборка фронтенда

### Требования

- Node.js;
- npm.

### Установка зависимостей

```bash
git clone <repository-url>
cd diplom/frontend
npm ci
```

### Режим разработки

```bash
npm run dev
```

Webpack будет отслеживать изменения исходного кода и пересобирать bundle:

```text
frontend/static/frontend/main.js
```

Команда не запускает отдельный frontend-сервер. Собранное приложение отдается через Django.

### Production-сборка

```bash
npm run build
```

## Взаимодействие с API

Для HTTP-запросов используется Axios.

Основные группы API:

```text
/auth          — регистрация, JWT и пользовательская сессия
/education     — курсы, модули и упражнения
/activities    — результаты и прохождение заданий
/user          — профиль и показатели пользователя
```

После успешной авторизации access token добавляется в заголовок:

```http
Authorization: Bearer <access-token>
```

## Работа с прогрессом

Клиент получает историю показателей пользователя и отображает ее с помощью Chart.js.

Поддерживаются следующие метрики:

```text
QER   — коэффициент эффективности чтения
BOFI  — широта поля зрения
VM    — визуальная память
VMWP  — визуальная память с учетом позиции
LM    — логическая память
AA    — коэффициент внимания
```

Для представления результатов используются линейные и радиальные диаграммы.
